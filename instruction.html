Инструкция по проекту (React + Vite)

Это руководство объясняет основные принципы работы современных сайтов (SPA - Single Page Application), как локально запускать проект, как собирать финальную версию и как публиковать её на хостингах вроде beCloud.

1. Как устроен наш сайт?

Сайт разрабатывается на TypeScript с использованием библиотеки React и сборщика Vite. На этапе разработки код представляет собой сложную структуру компонентов (папки components, pages, файлы .tsx).

Браузеры напрямую такой код не понимают. Поэтому перед публикацией сайт необходимо скомпилировать (собрать).

💡 Суть компиляции
Инструмент Vite берет сотни файлов проекта и сжимает их в компактный набор абсолютно обычных статических файлов (один index.html, оптимизированный JavaScript и CSS). В таком виде сайту больше не нужен сложный сервер (Node.js/PHP) — он готов к загрузке на любой базовый хостинг.

2. Основные команды (Терминал)

Все команды выполняются в терминале (в папке проекта PGATK Website).

Режим разработчика (Development)

npm run dev

Запускает локальный сервер для разработки. Все изменения в коде мгновенно отображаются в браузере. Эта команда используется только во время написания кода.

Сборка финальной версии (Production Build)

npm run build

Эта команда компилирует проект. В корне проекта появится папка dist (сокращение от distribution). Именно эту папку и нужно загружать на рабочий сервер beCloud.

Локальный просмотр финальной сборки

npm run preview

Если вы просто попытаетесь открыть файл dist/index.html двойным кликом в браузере, вы увидите пустой экран (из-за современных политик безопасности браузера).
Команда preview запускает мини-сервер, который позволяет вам посмотреть собранный в папке dist сайт в точности так, как он будет работать на реальном сервере.

3. Публикация на хостинг (beCloud / Apache / Nginx)

Сайт в папке dist состоит только из HTML, JS, CSS и картинок. Это означает, что он будет летать на самых дешевых и простых тарифах без необходимости поддержки PHP или Node.js.

Важный нюанс (Настройка роутинга)

Поскольку у нас одностраничное приложение (React Router), сервер при обновлении любой страницы (например, /news) будет искать папку "news", которой физически не существует в dist.

Чтобы сервер не выдавал ошибку "404 Not Found", его нужно настроить так, чтобы на любой запрос он всегда отдавал главный файл index.html. А React Router уже сам разберется, какую страницу показать.

Для хостинга на Apache (beCloud обычно использует его):

Создайте текстовый файл с названием .htaccess в папке dist (и загрузите его на сервер вместе с файлами сайта) со следующим содержимым:

<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>

Для хостинга на Nginx:

В настройках сайта в конфигурационном файле nginx необходимо указать директиву try_files:

location / {
  root   /path/to/your/dist;
  index  index.html index.htm;
  try_files $uri $uri/ /index.html;
}

4. Краткий чек-лист для публикации

  • Откройте терминал и выполните npm run build.
  • Дождитесь появления сообщения об успешной сборке.
  • Откройте появившуюся папку dist.
  • Добавьте файл .htaccess (если сервер Apache).
  • Скопируйте всё содержимое папки dist в корень вашего сайта на хостинге beCloud через FTP или файловый менеджер панели управления.
